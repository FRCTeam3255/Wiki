## Kickoff

### Schedule

- re-watch game animation (remember to film)
- Feature importance List
    - Update point calculations in feature importance list
    - Review [Raking Legend](https://frcteam3255.github.io/Wiki/Build%20Season/Day%201.3%20-%20Feature%20Importance%20List/#ranking-legend)
    - Go down list and see if we want to change ranks
- Break
- Create list of subsystems the current year's robot needs (e.g. Shooter, Intake, etc.)
    - List popular design for each subsystem (e.g Pivot Intake, Linear Intake, etc.)
- Round 1 Archetype Creation
    - Break into round 1 groups and create frakenbot (must have 1 design for each subsystem)
        - Goal of frakenbot is to prove the that designs work together in a real robot
        - Make 3D mock cad
        - make presentation slides
- Round 1 Presentation
    - Present 3 frakenbot archetypes
    - Break frakenbots into subsystem designs list
        - In build season we only rank subsystem designs in round 2
    - Rank each subsystem design by (Complexity, Prior knowledge, future use)
        - These are different categories than build season, future use in offseason only
        - Follow all rankings criteria
    - Look at rankings consolidate to 1 frakenbot archetype
        - During build season we go to 2 archetypes
- Round 2
    - 1 big group
    - Create Mock CAD on projector with all subsystem decisons, confirm it works
    - Verify feature importance list satisfaction
    - Keep repearing until it does
- Department Split
    - Software
        - Software lists out states, guess motor counts, define motor categories, assign motors to states, create issues by clustered categories, protobot skeleton code, assign buttons
    - Mechanical
        - Skip protobot for offseason
        - *2D robot sketches
